# GPS サテライトコンパス — 計画書

Arduino Nano Every を使った GPS コンパス。GPS の COG（対地針路）をメインの方位ソースとし、
GPS が使えない／停船中のときは地磁気センサーコンパスに自動フォールバックする。
針路は超大型 7 セグ LED ×3 桁で常時表示し、16×2 LCD にセーリング向けの諸情報を表示する。

最終的には JLCPCB に基板、JLC3DP にケースを発注してデッキ据え付け型の計器に仕上げる。

---

## 1. システム構成

```
                        ┌──────────────────────────┐
  単3×8 (12V) ─────┬───▶│ Arduino Nano Every        │
                   │    │  (ATmega4809, VIN=12V)    │
                   │    └──┬────┬────┬────┬────┬───┘
                   │       │    │    │    │    │
                   │    UART1  I2C  GPIO GPIO ADC
                   │       │    │    │    │    │
                   │  ┌────▼┐ ┌─▼──────┐ │    └── タクトSW ×3 (抵抗ラダー)
                   │  │GPS  │ │DS1307  │ │
                   │  │GT-  │ │RTC     │ │  ┌──────────────────┐
                   │  │502  │ ├────────┤ └─▶│ LCD 16×2 (4bit)  │
                   │  │1PPS─┼▶│BM1422  │    └──────────────────┘
                   │  └─────┘ │地磁気  │
                   │          └────────┘    ┌──────────────────────┐
                   └── 12Vセグメント電源 ──▶│ 7セグ 2.3" ×3桁      │
                                            │ TBD62083(セグ側sink) │
                                            │ 2SA1015+2SC1815(桁側)│
                                            └──────────────────────┘
```

- **方位ソース**: AUTO（既定）= GPS Fix あり かつ SOG ≥ しきい値（初期値 1.0kt）なら COG、
  それ以外は地磁気センサー。設定で GPS 固定 / MAG 固定も選択可。
- **時刻**: GPS 時刻（+1PPS）を正とし、DS1307 RTC に同期。GPS 未受信時は RTC で継続。

## 3. 電源設計

- 単3×8 = 約 12V（アルカリ新品時 ~12.8V、終止 ~8V）
- **12V レール**: 7セグのセグメント電源（330Ω 経由）、および Nano Every VIN
- **5V レール**: Nano Every 内蔵レギュレータ出力 → LCD、GPS、DS1307、BM1422 モジュール、TBD62083 ロジック
- I2C は 5V バスで統一（BM1422 モジュールはレベル変換内蔵、DS1307 は元々 5V 品）
- 消費電流見積り: 7セグ（1/3 ダイナミック、平均 5 セグ点灯）≈ 55mA @12V、
  MCU+GPS+LCD ≈ 80〜100mA @5V → 合計 12V 換算で 約 100〜120mA → アルカリで実用 15h 前後

## 4. 7 セグ駆動（3 桁ダイナミック点灯）

- A-2301SRD-A はアノードコモン。セグメント側（カソード）を TBD62083 でシンク、
  桁コモン（アノード）を 12V から 2SA1015（PNP）でハイサイドスイッチ。
- 2SA1015 のベースは 12V 側にあるため、MCU から 2SC1815（NPN）+ 1kΩ で引き落として駆動。
- セグメント電流 ≈ 12mA、1 桁ピーク ≈ 7 セグ × 12mA ≈ 84mA < 2SA1015 の 150mA で余裕あり。
- 桁切替 ≥ 200Hz（タイマ割り込み）でチラつきなし。DP は針路表示に不要のため未配線。
- 針路が地磁気ソースのときは表示を周期的に短時間ブリンクさせてソースを区別（LCD にも明示）。

## 5. ピンアサイン（Nano Every）

| ピン | 接続先 | 備考 |
|---|---|---|
| D1 (RX1) | GPS TX | 9600bps NMEA。GPS 出力は実測 H≈2.8V で 5V AVR の Vih 不足 → **2SC1815 反転バッファ経由**、RX ピン(PC5)の INVEN で論理を戻す。※Nano Every は D1=RX1/D0=TX1 |
| D0 (TX1) | GPS RX | 設定コマンド用（1kΩ で 1:2 分圧して 3.3V 化）。未使用でも可 |
| D2 | GPS 1PPS | 2SC1815 反転バッファ経由、FALLING エッジ割り込みで受ける |
| D3–D8 | LCD RS, E, D4–D7 | 4bit モード |
| D9–D11 | 7セグ桁選択 DIG1–3 | 2SC1815 ベースへ（1kΩ） |
| D12, D13, A0–A3, A6 | 7セグ a–g | TBD62083 入力（アクティブ H） |
| A4 / A5 | I2C SDA / SCL | DS1307 + BM1422AGMV（5V バス。プルアップは各モジュール搭載分を利用） |
| A7 | ボタンラダー | MODE=直結, A=47k, B=94k(47k×2)、47k プルダウン |

→ 22 GPIO を全部使い切る構成。ピン不足のためボタンは ADC ラダー読みにした。

## 6. 機能仕様

### 7 セグ（常時）
- 現在針路 000–359（°）。ソース切替時・起動時は全桁テストパターン。

### ボタン
- **MODE**（1 個目）: 短押しで LCD モード巡回、長押しでそのモードの設定に入る
- **A / B**（2・3 個目）: モードごとの操作（スタート/ストップ、＋/−、リセット等）

### LCD モード（MODE 短押しで巡回）
1. **NAV** — SOG(kt) / COG / 方位ソース(GPS・MAG) / 衛星数・Fix 状態
2. **RACE TIMER** — レーススタート用カウントダウン（既定 5:00、A=開始/停止、B=分同期(SYNC)）。
   0 到達でそのまま経過時間カウントアップに移行。通常ストップウォッチとしても使用可
3. **LOG** — 積算距離(nm)・最高速・平均速。B 長押しでリセット
4. **CLOCK** — 時刻（GPS/RTC 同期状態表示）・日付・タイムゾーン
5. **POSITION** — 緯度・経度
6. **STEER** — 目標針路を設定し、現在針路との差を「◀ 15」のように左右表示（ヘディングロック）
7. **SETUP** — 方位ソース (AUTO/GPS/MAG)、AUTO の SOG しきい値、磁気偏差（真方位/磁方位の統一）、
   タイムゾーン、地磁気キャリブレーション（水平で 1 回転）

### 方位処理
- COG は低速時に暴れるため円環平均でスムージング（更新 2〜4Hz）
- 地磁気は水平前提（BM1422 は 3 軸磁気のみ、傾斜補正なし）→ 取付は水平を前提とし、制約として明記
- 磁気偏差（日本近海で概ね西偏 7〜10°）を設定値で補正し、GPS COG（真方位）と基準を統一

## 8. 開発ステップ

- **Phase 1: ブレッドボード単体確認**（それぞれ最小スケッチを `firmware/tests/` に保存）
  1. LCD 表示 → 2. GPS NMEA エコー（+CH340E で PC 直結確認） → 3. I2C スキャン・RTC・地磁気読み
  → 4. 7セグ 1 桁静的点灯 → 5. 3 桁ダイナミック → 6. ボタンラダー
- **Phase 2: 統合ファームウェア** `firmware/compass/` — モジュール分割:
  `gps` / `heading`（ソース選択・平滑・偏差補正）/ `display7seg`（タイマ割込）/ `ui`（LCD+ボタンのモード機械）/
  `racetimer` / `log` / `settings`（EEPROM 保存）
- **Phase 3: 実地テスト** — 徒歩・車・可能なら艇上で COG↔MAG 切替とスムージングを調整
- **Phase 4: 製品化** — KiCad で回路図・基板（`hardware/`）→ JLCPCB 発注、
  ケース 3D モデル（`case/`）→ JLC3DP 発注。7セグ窓・LCD 窓・ボタン・防滴を考慮
